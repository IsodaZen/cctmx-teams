# Tmux Review Worker

このスキルは、ワーカーペインの出力を取得し、成果物をレビューします。

## 概要

リーダーがワーカーの作業完了後、成果物を確認・評価するために使用します。

## 使用タイミング

- ワーカーがタスク完了を報告したとき
- 定期的な進捗確認時
- ワーカーの出力を確認したいとき

## 動作

1. ワーカーペインの出力をキャプチャ（スクロールバック含む）
2. 一時ファイルに保存
3. 出力を表示
4. レビューチェックリストを提示

## 前提条件

- `/tmux-worker` スキルでワーカーペインが作成されていること
- `.claude/worker-info` にワーカーペイン情報が保存されていること

## 使用方法

```
/tmux-review
```

スキルを実行すると、以下が自動的に行われます:

1. ワーカーペインの出力をキャプチャ（過去3000行）
2. 出力を一時ファイルに保存（タイムスタンプ付き）
3. 出力内容を表示
4. レビューチェックリストを提示

## レビューチェックリスト

スキル実行後、リーダーは以下のチェックリストで成果物を評価します:

### 共通チェック項目

- [ ] タスクの目的が達成されているか
- [ ] 要件がすべて満たされているか
- [ ] 制約条件が守られているか
- [ ] コードの品質（可読性、保守性）は適切か
- [ ] エラーハンドリングが適切に実装されているか
- [ ] TypeScript型エラーがないか
- [ ] ESLintエラーがないか
- [ ] ビルドが成功するか

### タスクタイプ別チェック

#### 機能追加の場合

- [ ] 既存機能への影響がないか
- [ ] ドキュメントが更新されているか
- [ ] 型定義が適切か
- [ ] UIの一貫性が保たれているか

#### バグ修正の場合

- [ ] 根本原因が解決されているか
- [ ] 同様のバグが他にないか
- [ ] 回帰テストが追加されているか
- [ ] 副作用がないか

#### リファクタリングの場合

- [ ] 動作が変わっていないか
- [ ] パフォーマンスが劣化していないか
- [ ] すべてのテストが通るか

## レビュー結果の判断

### 合格（マネージャーに報告）

- すべてのチェック項目をクリア
- コードの品質が基準を満たしている
- 問題が見つからない

### 不合格（修正指示）

- チェック項目で問題が見つかった
- コードの品質が基準を下回っている
- 要件が満たされていない

不合格の場合は、具体的な修正指示を作成し、ワーカーに再度指示を送ります。

## 注意事項

- レビューは妥協せず、批判的に実施してください
- 「動けばいい」ではなく、品質を追求してください
- 問題が見つかった場合は、必ず具体的な修正指示を作成してください

## トラブルシューティング

### ワーカーペイン情報が見つからない

**原因**: `/tmux-worker` スキルを実行していない

**対処法**:

1. `/tmux-worker` スキルを実行
2. ワーカーペイン情報が `.claude/worker-info` に保存される
3. `/tmux-review` スキルを再実行

### 出力が空

**原因**: ワーカーがまだ作業を開始していない

**対処法**:

- ワーカーに指示を送信してから、しばらく待ってから再実行
