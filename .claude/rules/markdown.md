# Markdown コードスタイルガイド

このプロジェクトのMarkdownファイルは、以下のスタイルガイドに従ってください。

## 基本ルール

- ATXスタイルのヘッダーを使用（`#`、`===`ではなく）
- 可能な限り120文字以内に収める
- 言語指定付きのフェンスコードブロックを使用
- コードブロックの前後に空行を含める

## ヘッダー

```markdown
# レベル1ヘッダー

## レベル2ヘッダー

### レベル3ヘッダー
```

## コードブロック

言語を明示的に指定：

````markdown
```bash
#!/bin/bash
echo "例"
```

```json
{
  "key": "value"
}
```
````

## リスト

- 箇条書きリストには `-` を使用
- 番号付きリストには `1.` を使用
- ネストする場合は2スペースインデント

## リンク

- 外部リンク: `[リンクテキスト](https://example.com)`
- 内部リンク: `[リンクテキスト](./path/to/file.md)`
- 相対パスを使用

## 強調

- 太字: `**太字テキスト**`
- イタリック: `*イタリックテキスト*`
- コード: `` `インラインコード` ``

## 日本語と英語の混在

- 日本語と英語の間にスペースは不要
- 技術用語やコマンド名は英語のまま使用可能
